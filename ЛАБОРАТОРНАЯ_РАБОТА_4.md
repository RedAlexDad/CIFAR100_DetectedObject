## ❓ Контрольные вопросы

### 1. Что такое перенос обучения?

**Перенос обучения (Transfer Learning)** — метод машинного обучения, при котором модель, обученная на одной задаче, используется для решения другой связанной задачи.

**Преимущества:**
- Меньше данных для обучения
- Быстрее обучение
- Лучшее качество на малых данных

---

### 2. Архитектура предобученной модели

#### ResNet20 (чётный вариант)

```
Input (32×32×3)
  ↓
Conv1 (64, 3×3, stride=1)
  ↓
Residual Block 1 × 3
  ↓
Residual Block 2 × 3
  ↓
Residual Block 3 × 3
  ↓
Global Average Pooling
  ↓
Fully Connected (3 класса)
```

**Ключевая особенность:** Skip-connections для борьбы с затуханием градиента

#### MobileNetV2 (нечетный вариант)

```
Input (32×32×3)
  ↓
Initial Conv (32, 3×3, stride=2)
  ↓
Inverted Residual Blocks × 17
  ↓
Global Average Pooling
  ↓
Fully Connected (3 класса)
```

**Ключевая особенность:** Inverted residuals + depthwise separable convolutions

---

### 3. Что такое fine-tuning? Что такое заморозка весов?

**Заморозка весов** — блокировка обновления параметров модели во время обучения:
```python
for param in model.parameters():
    param.requires_grad = False
```

**Fine-tuning** — дообучение предобученной модели:
1. Замораживаем backbone (основную часть)
2. Обучаем классификатор
3. Размораживаем последние слои
4. Дообучаем всю модель с меньшим LR

---

### 4. Метод оптимизации Adagrad

**Adagrad** — адаптивный градиентный спуск с индивидуальным LR для каждого параметра.

**Формула:**
```
θ_{t+1} = θ_t - (η / √(G_t + ε)) ⊙ g_t
```

**Плюсы:**
- ✅ Не требует настройки LR
- ✅ Хорошо для разреженных данных

**Минусы:**
- ❌ LR может стать слишком маленьким
- ❌ Накопление градиентов не ограничено

---

### 5. Метод оптимизации RMSProp

**RMSProp** — модификация Adagrad с экспоненциальным затуханием.

**Формула:**
```
E[g²]_t = 0.9 × E[g²]_{t-1} + 0.1 × g_t²
θ_{t+1} = θ_t - (η / √(E[g²]_t + ε)) ⊙ g_t
```

**Плюсы:**
- ✅ Не даёт LR упасть до нуля
- ✅ Хорошо для нестационарных задач

**Минусы:**
- ❌ Требует настройки гиперпараметров

---

### 6. Метод оптимизации Adam

**Adam** — комбинация Momentum и RMSProp.

**Формула:**
```
m_t = β₁ × m_{t-1} + (1-β₁) × g_t          # Первый момент
v_t = β₂ × v_{t-1} + (1-β₂) × g_t²         # Второй момент
θ_{t+1} = θ_t - (η / √(v̂_t + ε)) ⊙ m̂_t   # Обновление
```

**Плюсы:**
- ✅ Сочетает преимущества Momentum и RMSProp
- ✅ Минимум гиперпараметров
- ✅ Хорошо работает по умолчанию

**Минусы:**
- ❌ Может сходиться к локальному минимуму
